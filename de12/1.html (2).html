git、htmlに関して

1　gitに関して
gitとはプログラムのソースコードなどの変更履歴を記録・追跡するための分散型バージョン管理システムである。
簡単に言うとファイルのバージョン管理が簡単に行うことができるツールである。
gitで管理しているファイルであれば、コンピューター上でのファイル管理が可能なため、編集前のファイルを残したまま、新しく編集したファイルの保存が可能。
gitが分散型バージョン管理システムといわれる所以
  • 古いバージョンに簡単に戻すことが可能
  • 新旧のファイルの一元管理が可能
  • 編集した履歴を複数人で共有が可能
  • 複数人で修正した部分を一つに統合することが可能


　　gitはCUIツールである。プログラマーが利用する文字だけで黒い背景の画面がCUI仕様の画面である。



CUIとGUIの違い
CUI キャラクターユーザーインターフェース
GUI グラフィカルユーザーインターフェース
キャラクターは文字、グラフィカルは画像、図面といった意味。
つまり違いは、インターフェース(境界面、接続面)で、文字を使うのか画像を使うのかの違いである。



　　CUI

GUI

決定的な違いは操作感。CUIはキーボードを入力し、コマンドを入力して動作をコンピューターに支持するというイメージだが、一方GUIは表示されているボタン、アイコンをマウスでクリックするだけで作動するイメージ。
つまり、操作するのにキーボードで入力を要するのがCUI、マウスで操作が完結するのがGUIであるといえる。



gitとgithubの違い

git 誰がいつどのように編集したかを把握可能なバージョン管理システム。

github gitの仕組みと連携し、他のユーザーとやり取りしやすくするwebサービス。

github = GUI ↔　git = CUI
